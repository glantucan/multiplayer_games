Dumb server

Doesn't sync players state when a new player connects. That's doable but I decided not to follow this approach.

It's ok if the game doesn't start until all players have joined, and no new players are allowed until the game ends. Else every gameobject state has to be exactly synchronized upon connection on the new client. As the server is supposed to be completely dumb that data would need to be taken form another client.

Also, every client acts as an independent thread running the same code. Random state changes and timer events need to be synchronized on all clients. That's another can of worms.

Maybe one of the clients, the one with the lowest lantency, could be considered the authority regarding timer events and random state changes. But maintaining the clients in sync can become extremely complex.

Another option is to run all timer events and random state chenge login on the server. Less latency, but not prettier.
